# 🏋️ wgym (Мобильный фитнес-трекер)

**wgym** — это современное Android-приложение для учёта питания, тренировок и анализа прогресса. Оно помогает вести дневник калорий, создавать тренировочные шаблоны, проводить тренировки и отслеживать статистику прямо на вашем смартфоне. Приложение построено на чистом **Kotlin** с использованием **Jetpack Compose** для декларативного UI, **Room** для локальной базы данных, **Dagger Hilt** для внедрения зависимостей и архитектуры **MVVM** с **StateFlow** для реактивного управления состоянием.

---

## 🎯 О проекте

### Зачем был создан этот продукт?

Современный ритм жизни требует удобного инструмента для контроля питания и физической активности, который всегда под рукой. **wgym** объединяет в себе все необходимые функции: учёт продуктов и приёмов пищи, управление тренировочными шаблонами, проведение тренировок с записью результатов и наглядную статистику — всё в одном приложении. Проект разработан как пример комплексного мобильного решения с продуманной архитектурой, масштабируемостью и тестируемостью.

### Какую проблему он решает?

Приложение заменяет разрозненные заметки, таблицы и бумажные дневники единой цифровой системой, которая автоматически подсчитывает калорийность и БЖУ, сохраняет историю тренировок, показывает динамику веса и даёт визуальную обратную связь по прогрессу. Все данные хранятся локально на устройстве — никакой зависимости от облачных сервисов и интернета.

### Ключевые преимущества

1. **Полная локальность и приватность** — все данные остаются только на вашем телефоне.
2. **Интуитивный интерфейс** на Jetpack Compose — быстрая и отзывчивая навигация.
3. **Гибкое управление питанием** — продукты, приёмы пищи, автоматический расчёт КБЖУ.
4. **Мощный тренировочный модуль** — упражнения, шаблоны, проведение тренировок с историей.
5. **Визуальная статистика** — графики калорийности за неделю и месяц, прогресс по тренировкам.
6. **Современная архитектура** — MVVM, Clean слои, инъекция зависимостей, реактивные потоки.

## 🚀 Установка и запуск

Чтобы запустить приложение на своём устройстве или эмуляторе:

### 1. Клонируйте репозиторий

```bash
git clone https://github.com/darya-zaytseva/wgym-android.git
cd wgym-android
```

### 2. Откройте проект в Android Studio

Рекомендуется **Android Studio Flamingo** или новее.

### 3. Настройте базу данных (предварительные данные)

Приложение использует Room. Для демонстрации вы можете:
1. Запустить приложение «из коробки» — Room создаст пустую БД.
2. Или загрузить подготовленный дамп (файл `.db`) в папку `assets` и скопировать его при первом запуске (реализовано через `createFromAsset()`).

В текущей реализации БД создаётся автоматически при первом запуске.

### 4. Запустите приложение

Выберите эмулятор (например, Pixel 6 API 33) или подключите реальное устройство с отладкой по USB. Нажмите **Run** (зелёная стрелка).

> ⚠️ Убедитесь, что у вас установлены все необходимые SDK и инструменты сборки (Android Gradle Plugin 8.0+).

## ✨ Возможности

### 🍎 Управление питанием

1. **Продукты**: добавление, редактирование, удаление, поиск по названию. Хранятся: название, калорийность, белки, жиры, углеводы.
2. **Приёмы пищи**: создание приёма (завтрак, обед, ужин, перекус) с привязкой к дате и времени, добавление продуктов с указанием порции (в граммах или штуках). Автоматический подсчёт суммарной калорийности и БЖУ.
3. **История**: просмотр всех приёмов пищи за день, возможность редактировать или удалять записи.
4. **Дневник калорий**: календарная сетка с отображением суммарной калорийности за день, цветовая индикация превышения нормы.

### 🏋️ Управление тренировками

1. **Упражнения**: создание, редактирование, удаление, поиск. Для каждого упражнения указывается название, описание, группа мышц, URL картинки (опционально).
2. **Шаблоны тренировок**: создание шаблона из списка упражнений с указанием количества подходов, повторений и веса. Можно редактировать и удалять готовые шаблоны.
3. **Проведение тренировки**: выбор шаблона, запуск таймера (или ручной учёт времени), внесение фактически выполненных подходов, повторений и весов. Сохранение результата в историю.
4. **История тренировок**: список завершённых сессий с деталями, возможность просмотра выполненного объёма.

### 📊 Статистика и аналитика

1. **Питание**: график суммарной калорийности за последние 7 дней и за текущий месяц (столбчатая диаграмма). Сравнение с рекомендуемой нормой.
2. **Тренировки**: количество тренировок за месяц, общий объём (тоннаж), прогресс по каждому упражнению (график изменения рабочих весов).
3. **Вес тела**: возможность фиксировать вес, график динамики за выбранный период.
4. **Главный экран**: сводка за сегодня — калории, белки/жиры/углеводы, количество тренировок за неделю.

### 👤 Профиль и авторизация

1. Регистрация.
2. Просмотр сводной статистики.
3. Гостевой режим с ограниченным функционалом.

## 📖 Использование

### Первый запуск

1. При первом запуске откроется экран приветствия. Выберите **Вход** или **Регистрация**.
2. Заполните обязательные поля (имя, email, пароль). Пароль хешируется локально.
3. После успешного входа вы попадёте на **главный экран** с дневной сводкой.

### Основные экраны

| Экран | Описание |
|-------|----------|
| **Главный** | Сводка за сегодня (калории, БЖУ, количество приёмов пищи, статус тренировки) |
| **Питание** | Вкладка «Дневник» (список приёмов пищи за день) и «Продукты» (поиск и управление продуктами). Кнопка «+» для добавления приёма пищи |
| **Тренировки** | Вкладка «Шаблоны» (список и управление), «Выполнить» (запуск тренировки по шаблону), «История» (список завершённых) |
| **Упражнения** | Полный список упражнений, поиск, добавление/редактирование через диалог |
| **Статистика** | Переключение между вкладками «Калории», «Тренировки», «Вес». Интерактивные графики с выбором периода |
| **Профиль** | Редактирование данных, смена целевой калорийности, просмотр общей статистики, выход |

### Работа с данными

1. Все изменения сохраняются мгновенно в локальную базу через Room.
2. Для добавления/редактирования сущностей используются диалоговые окна или отдельные экраны (в зависимости от сложности).
3. Реактивное обновление UI обеспечивается через `StateFlow` во `ViewModel` — достаточно изменить данные в БД, и экран автоматически перерисуется.

## 🏗 Архитектура и разработка

Проект построен на **чистой архитектуре** с тремя слоями:

1. **Data Layer**: реализация репозиториев, Room DAO и сущности, кэширование.
2. **Domain Layer**: бизнес-логика (интеракторы / use cases), модели предметной области, интерфейсы репозиториев.
3. **Presentation Layer**: ViewModel, State (UI-состояния), Compose-экраны, навигация.

### Внедрение зависимостей

**Dagger Hilt** используется для инъекции:
1. Репозиториев во ViewModel (через `@HiltViewModel`).
2. DAO в репозитории.
3. Use Cases в ViewModel (при необходимости).

### Реактивность

1. ViewModel содержит `MutableStateFlow` для состояния экрана и обновляет его через `update` методы.
2. UI подписывается на `StateFlow` через `collectAsState()`.
3. Одноразовые события (например, навигация, показ тоста) реализованы через `SharedFlow` в ViewModel и обрабатываются в Compose-экранах.

### Навигация

Используется `NavHost` с дефолтным `ComposeNavigation`. Граф переходов соответствует диаграмме вариантов использования (авторизация, питание, тренировки, статистика, профиль).

### Работа с базой данных

Room-сущности: `User`, `Product`, `Meal`, `MealProduct` (связующая), `Exercise`, `WorkoutTemplate`, `TemplateExercise` (связующая), `WorkoutSession`, `SessionExercise`, `WeightEntry`. Миграции версий выполняются через `Room.databaseBuilder().addMigrations()`.
